## [2] API 구현 (임포트 완료 후)

```
C:\project\냉파튜브\기획서.md 와 C:\project\냉파튜브\프롬프트_설계.md 를 읽어라.

그다음 스펙대로 구현해줘:
1. POST /api/recommend — 재료 배열을 받아 Claude API(haiku)를 호출.
   시스템 프롬프트는 프롬프트_설계.md의 완성본을 그대로 사용.
   입력 정제(재료당 20자, 최대 15개, 특수문자 제거), JSON 스키마 검증,
   실패 시 1회 재시도, 재실패 시 문서의 폴백 목록 반환.
2. GET /api/videos?dish= — YouTube Data API v3 search.list
   (쿼리 "{dish} 레시피", order=viewCount, 상위 3개).
   요리명 정규화(trim/공백통일/NFC) 후 캐시 키 yt:{요리명}로
   Upstash Redis에 TTL 7일 캐싱. 캐시 히트 시 API 호출 금지.
3. 프론트의 목 데이터를 실제 API 호출로 교체.
4. 환경변수: ANTHROPIC_API_KEY, YOUTUBE_API_KEY,
   UPSTASH_REDIS_REST_URL, UPSTASH_REDIS_REST_TOKEN — .env.example 생성.

금지: 기획서 2.5 비목표 섹션의 기능(회원가입, 유통기한, 사진인식,
쿠팡, 카톡)은 어떤 형태로도 만들지 마라.
완료 기준: npm run dev 후 재료 입력→추천→영상 재생까지 로컬 완주.
```
