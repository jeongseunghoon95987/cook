# 냉파튜브 (가칭) — 기획 문서

> 냉장고 재료를 입력하면, 만들 수 있는 요리와 그 요리의 유튜브 레시피 영상을 바로 골라주는 웹 서비스

- 작성일: 2026-07-09
- 상태: 기획 확정, M0 착수 전
- 포지셔닝: "냉장고 관리 앱"이 아니라 **"재료 → 오늘 볼 요리 영상 큐레이션"**

---

## 1. 배경 및 시장 검증

### 1.1 검증된 수요
- 동일 컨셉 앱이 2018년(밥타임)부터 2026년 현재까지 계속 신규 출시됨
  → 문제(오늘 뭐 해먹지 + 재료 낭비)가 실재한다는 증거
- 경쟁 앱: 냉장고파먹기(냉파), eggg.ai, 요리보고, 냉쉐프, 냉장고 셰프, 냉장고 털기 등

### 1.2 시장의 공백 (차별화 포인트)
- 기존 앱 전부 텍스트 레시피(자체 DB 또는 LLM 생성) 제공
- **유튜브 영상 큐레이션을 하는 곳이 없음** — 실제 한국인의 요리 행동은
  "영상 틀어놓고 따라하기"인데 이 행동과 연결한 서비스 부재
- 기존 앱들의 공통 실패 지점: 재료 등록이 귀찮아 유저가 조기 이탈
  → 냉파튜브는 장부 관리를 요구하지 않음. "지금 있는 거 3개만 입력"의
    일회성 사용만으로 가치가 나오는 구조

### 1.3 솔직한 리스크
- 큰돈 벌 확률은 낮음. 첫 "기획→출시→측정" 완주 프로젝트로 가치 판단
- 진입장벽 낮음 — 잘되면 기존 앱이 유튜브 기능을 빠르게 모방 가능 (해자 없음)
- 기대값: 수익화 확률 20% / 학습 가치 100%

## 2. MVP 스펙 (v0.1)

### 2.1 코어 플로우 (이것만 만든다)
1. **재료 입력** — 텍스트 자유입력 + 자주 쓰는 재료 30종 원터치 칩
   (계란, 대파, 김치, 두부, 양파, 돼지고기...). 최소 2개부터 추천 가능.
   로그인 없음. localStorage 저장.
2. **요리 추천** — Claude API(Haiku)에 재료 전달 → JSON으로 요리 8개.
   "지금 바로 가능"(4) / "1~2개만 더 사면 가능"(4) 두 그룹.
   각 요리에 난이도·소요시간 태그.
3. **영상 카드** — 요리 선택 시 YouTube Data API search.list
   (쿼리: "{요리명} 레시피", 조회수순) 상위 3개를
   썸네일·채널명·조회수·길이 카드로. 클릭 시 인라인 임베드 재생.

### 2.2 기술 스택 (2026-07-09 변경: Laravel → Next.js)
- 프레임워크: Next.js (App Router) — API Routes로 백엔드 겸용
- 배포: Vercel 무료 티어 (HTTPS/도메인/배포 자동, 서버 관리 없음)
- 캐시: Upstash Redis (또는 Vercel KV) — DB 없이 시작, MariaDB 세팅 제거
- LLM: Claude API (claude-haiku 계열, 비용 절감)
- 영상: YouTube Data API v3 (일 쿼터 10,000유닛, search 1회 = 100유닛)
- 변경 근거: 승부처가 빠른 출시 + SEO(v0.2 재료조합 랜딩의 SSG/ISR)이며
  Claude Design 산출물(React)을 그대로 이식 가능

### 2.3 엔드포인트 (2개뿐)
| 메서드 | 경로 | 역할 |
|---|---|---|
| POST | /api/recommend | 재료 배열 → LLM 호출 → 요리 8개 JSON (Route Handler) |
| GET | /api/videos?dish= | 유튜브 검색 (요리명 단위 7일 캐싱) (Route Handler) |

### 2.4 캐싱 설계 (유튜브 쿼터 방어 — 필수)
- Upstash Redis 키: `yt:{정규화된 요리명}` / 값: 영상 3개 JSON / TTL 7일
- 같은 요리명은 7일간 API 재호출 없이 캐시 반환
- 요리명이 LLM 출력이라 표기가 흔들릴 수 있음 → 요리명 정규화(트림/공백통일) 후 키로 사용

### 2.5 비목표 (v0.1에서 만들지 않는 것)
- 앱, 회원가입, 자체 레시피 DB
- 유통기한 알림, 사진 인식(비전), 쿠팡 파트너스, 카톡 알림
- 전부 v0.2 이후 후보. **지표가 나오기 전 착수 금지**

### 2.6 측정 지표 (GA4 이벤트 4개)
1. 재료 입력 완료율 (방문 → 추천 요청)
2. 요리 카드 클릭률
3. 영상 클릭률(재생)
4. 7일 재방문율

### M2 — D+14: 공개
- [ ] 자취/요리 커뮤니티 2~3곳 공유
- [ ] 1주일 지표 수집

### Go / Kill 판정 — D+21
- 조건: 방문자 최소 100명 확보 후
- **Go**: 입력 완료율 30%↑ AND 영상 클릭률 15%↑
  → v0.2 진행 (SEO 랜딩, 공유 기능, 애드센스)
- **Kill**: 미달 시 원인(유입 vs UX) 판별 후 1회 수정, 재미달이면 종료
- 이 기준은 사전에 못 박은 것 — 사후 변경 금지

## 4. v0.2 이후 로드맵 (Go 판정 시에만)

1. SEO 랜딩 — "OO+OO로 만드는 요리" 재료 조합별 정적 페이지 (검색 유입구)
2. 공유 기능 — 추천 결과 URL 공유
3. 애드센스 배너 (결과 화면 하단 1개)
4. 쿠팡 파트너스 — "부족한 재료" 구매 링크 (맥락형 커머스, 기대 수익 주력)
5. 사진 인식 — 냉장고 사진 → 비전 LLM 재료 추출 (바이럴용 와우 기능)
6. 유통기한 알림 — 웹푸시 → 카카오 '나에게 보내기' 연동

## 5. 수익 모델 메모
- 애드센스: 트래픽 임계점(월 수만 PV) 전까진 보조
- 쿠팡 파트너스가 본명 — "고추장이 없네요 → 최저가" 는 광고가 아니라 기능
- 참고: 경쟁 앱 '냉파'도 쿠팡 연동을 이미 함 → 유튜브 큐레이션이 유일한 차별점임을 잊지 말 것

## 6. Claude Design 프롬프트

아래 원문을 Claude Design에 그대로 붙여넣기:

```
모바일 우선 반응형 웹서비스 "냉파튜브"의 UI를 디자인해줘.
한 줄 컨셉: 냉장고 재료를 입력하면 만들 수 있는 요리를 추천하고,
그 요리의 유튜브 레시피 영상을 바로 보여주는 서비스.

화면 3개:

1) 홈(재료 입력): 상단에 서비스 로고와 "냉장고에 뭐 있어요?" 헤드라인.
재료 텍스트 입력창(입력하면 칩으로 변환) + 그 아래 자주 쓰는 재료
원터치 칩 그리드(계란, 대파, 김치, 두부, 양파, 돼지고기 등 30개,
카테고리 탭: 채소/육류/양념/기타). 선택된 재료는 상단에 칩으로 쌓임.
하단 고정 CTA 버튼 "요리 찾기 (재료 3개)".

2) 추천 결과: 두 섹션으로 구분 — "지금 바로 만들 수 있어요"(4개),
"1~2개만 더 사면 돼요"(4개, 부족 재료를 작은 배지로 표시).
각 요리 카드: 요리명, 난이도(쉬움/보통), 소요시간, 사용 재료 태그.
카드는 세로 리스트.

3) 요리 상세(영상): 선택한 요리명 헤더 + 유튜브 영상 카드 3개
(큰 썸네일, 채널명, 조회수, 영상 길이). 첫 카드 탭하면 인라인
플레이어로 재생. 하단에 "다른 요리 보기" / "재료 다시 고르기" 버튼.

상태 화면: 로딩(냄비 끓는 애니메이션 + "레시피 찾는 중..."),
재료 미선택 빈 상태, API 에러 상태.

톤앤매너: 밝고 식욕 돋는 따뜻한 색(크림 배경 + 토마토/당근 계열
포인트), 둥근 모서리, 큰 터치 타겟, 친근한 한국어 카피.
과한 장식 없이 요리 앱답게 깔끔하게. 광고 배너 영역을 결과 화면
하단에 1개 자연스럽게 예약해줘.
```
